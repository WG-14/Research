P1. Correct or explicitly disclose delayed-fill equity/MDD timing.

Preferred design change:

Implement a minimal event-time-aware accounting update so that delayed fills do not affect the signal candle close equity. At minimum:

* compute the current candle close equity using the pre-fill portfolio state when the fill reference occurs after the signal candle close;

* preserve deterministic behavior and do not break trade metrics.

However, prefer the actual timing fix over merely documenting the approximation.

P1. Clarify and enforce `missing_quote_policy` semantics.

Inspect `ExecutionTimingPolicy`, `resolve_execution_reference()`, `run_sma_backtest()`, `execution_reality_gate()`, and report generation.

Make `fail`, `skip`, and `warn` operationally distinct:

```text
fail:
  - missing execution quote creates failed execution evidence
  - scenario/candidate gate fails with a clear reason
  - reason example: quote_after_decision_signal_coverage_below_threshold or missing_quote_failed

skip:
  - no fill and no position update
  - execution attempt is recorded as skipped, not filled and not silently omitted
  - reason example: missing_quote_skipped
  - signal quote coverage still counts the signal event

warn:
  - no fallback to candle close unless a separate explicit fallback policy exists
  - execution attempt is recorded as skipped_with_warning or failed_with_warning
  - report warnings include missing_quote_warning
  - promotion-grade candidates should not pass if required quote coverage is below the configured threshold
```

Do not allow missing quote behavior to silently fall back to candle close under quote-aware policies. If no fallback is explicitly configured, missing quote means no executable reference.

Ensure signal quote coverage remains accurate even when an execution attempt is skipped. If current coverage is computed only from trades, add a separate execution-attempt metadata collection or make skipped attempts appear in execution metadata without affecting position or closed-trade metrics.
