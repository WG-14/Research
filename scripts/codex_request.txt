P0. Implement event-time-safe portfolio state for delayed fills, or fail closed when safe event-time accounting is not available.

Preferred design:

Introduce a minimal pending execution mechanism inside `run_sma_backtest()`:

* When a fill is created and `fill_reference_ts` is greater than the current candle close timestamp, do not immediately apply its portfolio effects.
* Store it as a pending fill event with:

  * fill object
  * side
  * fill_reference_ts
  * quantity
  * cash/qty delta
  * fee/slippage
  * entry/exit regime snapshots
  * accounting metadata needed to finalize the trade
* At the beginning of each candle iteration, before computing action eligibility for that candle, apply all pending fills whose `fill_reference_ts` is at or before the current decision/mark boundary chosen by the timing model.
* Action gating must use only effective portfolio state plus any explicitly tracked pending state. It must not treat a future BUY as current `qty > 0`.
* SELL logic must not be enabled by a BUY fill before that BUY’s `fill_reference_ts`.
* Equity/MDD marks must use portfolio state that is effective at that mark time, not state from future fills.
* Preserve deterministic results.

If implementing full pending-fill accounting is too invasive, add a fail-closed guard for promotion-grade evidence:

