## Required patch, if H2 is confirmed

Refactor `paper_execute()` so the unresolved-order gate runs before orderbook quote loading.

Target behavior:

```text
1. Validate paper execution config.
2. Resolve canonical market.
3. Open DB connection.
4. init_portfolio(conn).
5. evaluate_unresolved_order_gate(...).
6. If gate blocked:
   - log `[SKIP] unresolved order gate`
   - notify consistently with existing paper skip style
   - commit if needed
   - return None
   - do not fetch orderbook
   - do not call patched `_get_fill_price`
   - do not create orders/fills/trades
7. Only if gate is clear:
   - fetch orderbook / build quote context / resolve fill price
   - continue existing sizing, dedup, order recording, stress/immediate execution flow
```

Be careful with connection lifetime. If quote fetching now happens inside the DB `try/finally`, make sure the connection is closed on all returns and exceptions.

* mark partial fills as `FILLED`
