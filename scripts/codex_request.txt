Core problem to investigate:

The repository appears to contain two competing DB schema authorities:

* `src/bithumb_bot/db_schema.py` appears to define an older schema where `portfolio` uses `cash` and `qty`, and `trades` uses older accounting columns.
* `src/bithumb_bot/db_core.py` appears to define the current runtime schema where `portfolio` uses `cash_krw`, `asset_qty`, `cash_available`, `cash_locked`, `asset_available`, `asset_locked`, plus order/fill/repair/observation tables.
* Runtime code appears to use `db_core.ensure_db()`, but the old `db_schema.py` module may still allow stale scripts or experiments to create a structurally wrong database.

Do not solve this as a cosmetic cleanup. The goal is to make the repository more robust, reproducible, recoverable, and operational as an automated trading experiment system. Prioritize design-level prevention of the whole problem family over symptom patches.
