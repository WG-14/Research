## Design goal

Do not merely rename a variable in one place. Fix the metrics contract so every ratio with “drag” semantics is either:

1. computed with the same basis across layers, or
2. explicitly basis-tagged so operators cannot mistake one basis for another.

The preferred design is to add explicit basis fields everywhere the ratios are emitted.

For research metrics v2:

```text
cost_execution.fee_drag_ratio_basis = "traded_notional"
cost_execution.slippage_drag_ratio_basis = "traded_notional"
```

For runtime strategy performance, if the current denominator remains `abs(gross_pnl)`:

```text
summary.fee_drag_ratio_basis = "gross_pnl_abs"
```

If you decide to preserve the old runtime `fee_drag_ratio` field for backward compatibility, add a clearly named alias:

```text
fee_to_gross_pnl_ratio = same value as current fee_drag_ratio
fee_to_gross_pnl_ratio_basis = "gross_pnl_abs"
```

Then either:

* keep `fee_drag_ratio` with `fee_drag_ratio_basis="gross_pnl_abs"` for compatibility, or
* if all downstream callers can safely migrate, rename runtime `fee_drag_ratio` to `fee_to_gross_pnl_ratio`.

Prefer backward compatibility unless the repo already has an explicit fail-closed contract requiring a rename.

If runtime can compute traded notional safely from `trade_lifecycles`, then add:

```text
fee_drag_ratio = fee_total / total_traded_notional
fee_drag_ratio_basis = "traded_notional"
fee_to_gross_pnl_ratio = fee_total / abs(gross_pnl)
fee_to_gross_pnl_ratio_basis = "gross_pnl_abs"
```

Only do this if the data is genuinely available and the formula is deterministic. Do not invent notional from incomplete data.

## Implementation requirements

### 1. Add ratio basis fields to the research metrics contract

Update `MetricContractV2` / `CostExecutionMetrics` so `as_dict()` emits explicit basis fields.

Expected fields:

```text
fee_drag_ratio_basis
slippage_drag_ratio_basis
```

For current research metrics, the basis should be:

```text
"traded_notional"
```

If `fee_drag_ratio` or `slippage_drag_ratio` is `None` because there is no traded notional, the basis should still be explicit unless there is a strong reason not to. Recommended:

```json
{
  "fee_drag_ratio": null,
  "fee_drag_ratio_basis": "traded_notional",
  "slippage_drag_ratio": null,
  "slippage_drag_ratio_basis": "traded_notional"
}
```

Keep strict JSON compatibility.

### 3. Update operator/reporting outputs
